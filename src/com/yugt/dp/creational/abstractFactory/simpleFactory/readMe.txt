*定义:
简单工厂模式专门定义一个类来负责创建其它类的实例，被创建的实例通常都具有共同的父类。


*角色：
Factory：工厂角色
Product：抽象产品角色
ConcreteProduct：具体产品角色


*
简单工厂模式的关键在于：工厂方法是静态的


*
1.又称为静态工厂方法(Static Factory Method)模式
2.优点：工厂类含有必要的判断逻辑，客户端可以免除直接创建产品对象的责任（简单工厂模式通过这种做法实现了对责任的分割）
3.缺点：简单工厂模式通常使用静态工厂方法，子类无法覆盖父类的静态方法，造成工厂角色无法形成基于继承的等级结构


*
应用场合：
1.Java密码学（Cipher cp=Cipher.getInstance("DESede");//一种加密算法，见DESEncrypt.java）
